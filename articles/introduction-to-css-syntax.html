Introduction

This article will cover the basics of cascading style sheets (CSS) and how to apply them
to HTML documents. A cursory overview of CSS syntax will also be included.

Summary

This article will cover the basics and application of CSS. We will first learn what CSS is
and how it is structured. Then we will learn the various ways to integrate CSS with HTML.

What is CSS?

CSS (cascading style sheets) are documents that tell a browser how to present or style an
HTML document. Within CSS are rule sets (also called "rules") that provide the browser with
the information on how to display the HTML document that it is attached to. Rules can change
anything from font size to where certain elements are placed. So, while HTML is the framework
of the webpage, CSS is the accompanying instructions that arrange and design the webpage.

Basic Structure

CSS rules all follow the same basic structure: selector {property: value;}. Curly braces encase
every block of rules and each rule ends with a semi-colon. The "selector" is the part of
the HTML document that is affected by the rule. The "selector" can be an element (<p>), class
(<p class="XYZ">), or id (<p id="XYZ">). The "selector" tells the browser what part of the
document is affected by the following rule. The "property" is what you want to change about the
selected part of the document. This can be anything from font color to positioning. The "value"
how much is changed. The "value" can be measurements (em, pt, cm, %,), positioning (center, left, right),
color (color names or hex codes), or font family (serif, sanserif, Times, Helvetica). Below is an
example of a rule that changes the text in h1 to be red. 


            h1 { color: red;}

Selectors

Selectors are selective. A <p> "selector" will affect all <p> elements, a class selector will
only affect elements with that class, and an id selector will only affect elements with that id.
As stated above, a selector can be n element, class, or id. Each of these is inputted differently
into CSS. An element just uses its tag with out the "<" and >". So a "<p>" element would be entered
into CSS as "p". A class uses a "." (period) in front of the class name. So "<p class= XYZ>" would be
entered as ".XYZ". An id uses a "#" (pound sign) in front of its id name. So "<p id= XYZ>" would be
entered as "#XYZ". Examples of these are below. 


            p { color: red;}
        .XYZ {color: red;}
        #XYZ {color: red;}

Selector Disagreement

CSS keeps selectors from disagreeing by being specific about which element that rule affects. Class
and id selectors more specific than element selectors. This enables a classed <p> to be different from
a regular <p>. The rule below makes the classed <p> have blue text while all the other <p> have red text. 


            p {color: red;}
        .XYZ {color: blue;}

Rule Grouping and White Space

Multiple rules can affect the same element. Those rules can be listed out separately or bundled together
in the same block as shown below (this can also be done with selectors, but will be covered in the Advanced
CSS Selectors article). The browser ignores white space, so you can put as much space between rules as you want.
This is often done to allow for easier reading of the rules. 


        p {color: red;}
        p {text-align: center;}

        This is the same as below:

        p {color: red; text-align: center;}

        Which is the same as:

        p {color: red;
               text-align: center;}

The browser reads CSS so that rules towards the bottom of the CSS document take precedence over those at
the top. This is much like an amendment. Much like an amendment overrides those above it, a rule will
override rules above it. This resolves conflicts between rules as shown below.


            p {color: red;}
        p {color: blue;}

The text in the "<p>" element will be blue rather than red because the lower rule overrides the one above it.

Itegrating CSS with HTML

CSS can be applied to HTML in three ways: in-line, embedded, and external. In-line is done by applying
CSS with the "style" attribute as shown below.


            <p style="color: red; font-family: Helvetica;">Blah</p>

This type of CSS is useful as it will override embedded and external CSS, but it also makes it difficult
with maintaining the webpage as the rules are scattered throughout the HTML document.
Embedded CSS embeds CSS rules within the "<head>" of the document within a "<style>" element as shown below.


        <head>
        <style type="text/css" media="screen">
            p {color: blue;
               font-family: Helvetica;
            }
        </style>
        </head>

This way of integrating CSS allows one to affect the whole document rather than each individual element in
the HTML document. It also keeps everything in one place so as to aid with maintaining the webpage.
External CSS takes a CSS document and links it to HTML documents using the <link> element within the “<head>”
of the document as shown below.


        <head>
        <link rel=“stylesheet” href=“styles.css” type=“text/css” media=“screen”>
        </head>

Using external CSS allows one to use one CSS document on multiple HTML documents. This keeps the style
and look of the webpages of a document the same without having to type in the same rules over and over again.
